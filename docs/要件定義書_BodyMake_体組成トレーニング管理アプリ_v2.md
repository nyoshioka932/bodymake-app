# 要件定義書：BodyMake 体組成・トレーニング管理アプリ

**バージョン：** 2.0  
**更新日：** 2026年6月11日  
**対象ユーザー：** 個人利用を前提。ただしDB設計は将来の複数ユーザー化を考慮  
**更新方針：** 既存のローカルPC / SQLite中心設計を、スマホ中心・クラウドDB・Googleログイン・週次PDCA中心へ改訂する。

---

## 1. システム概要

BodyMakeは、体組成、摂取カロリー/PFC、消費カロリー、筋トレ記録、体型写真を統合し、週次PDCAを回すための個人向けフィットネス管理アプリである。

単なる記録アプリではなく、以下を実現する。

1. 体重・体脂肪率・体脂肪量・骨格筋量のトレンド把握
2. 摂取カロリー、消費カロリー、PFCの週次評価
3. 筋トレの部位別セット数と種目別成長の把握
4. 直近7日間の状態から、次に取るべき改善アクションを提示
5. スマホ中心で、外出先でも閲覧・筋トレ入力できる体験

---

## 2. 最終目的・MVP目的

### 2.1 最終目的

**筋肉量を維持または増加させながら、体脂肪を減らす。**

体脂肪率だけを追うのではなく、以下の複合指標で判断する。

- 体脂肪率
- 体脂肪量 kg
- 骨格筋量 kg
- 体重7日移動平均
- 摂取・消費カロリー収支
- PFC、特にタンパク質と脂質
- 部位別週間セット数
- 種目別成長、主に推定1RM

### 2.2 MVP完成条件

MVPは、以下が使える状態を完成条件とする。

1. Googleログインできる
2. Supabaseにユーザー別データを保存できる
3. オムロンCSV、Apple Health ZIP、Fitbit CSVを取り込める
4. 体組成・カロリー・PFC・筋トレのダッシュボードが見られる
5. スマホで筋トレ記録を入力できる
6. 週次サマリーが見られる
7. アラートと改善アクションが表示される
8. データ一覧から閲覧・削除・再取込導線が利用できる
9. CSV/JSON/取込ログをエクスポートできる
10. 使い方ガイドを参照できる
11. サンプルデータで動作確認できる

## 5. データソースと連携方式

### 5.1 体組成データ：オムロンCSV

| 項目 | 要件 |
|---|---|
| データソース | オムロン体組成計アプリ、KRD-608T2想定 |
| 連携方式 | CSV手動アップロード |
| アップロード端末 | PC・スマホどちらも対応 |
| 既存CSV | 手元にある前提 |
| 計測頻度 | 日によって朝だけ、夜だけ、複数回があり得る |
| 分析代表値 | 同一日の最も早い時刻の値 |
| 表示対象 | 体重、体脂肪率、体脂肪量、骨格筋率、骨格筋量 |
| その他取込 | 内臓脂肪、基礎代謝、BMI、体年齢は保存対象としてもよいが、MVP表示優先度は低い |

分析用の代表値は「朝」と断定せず、**日次代表値**として扱う。

```text
同一日の体組成レコードのうち measured_at が最も早いものを is_representative = true にする。
```

### 5.2 摂取カロリー・PFC：あすけん → Apple Health ZIP

| 項目 | 要件 |
|---|---|
| 記録アプリ | あすけん |
| 連携方式 | Apple Health ZIP手動エクスポート → アップロード |
| 対象データ | あすけん由来の栄養データのみ |
| 取込粒度 | 日単位 |
| 対象期間 | ZIP内の全期間 |
| 取込項目 | 摂取カロリー、タンパク質、脂質、炭水化物 |
| 食事単位 | MVPでは不要。将来検討 |

対象レコードタイプは以下を想定する。

- `HKQuantityTypeIdentifierDietaryEnergyConsumed`
- `HKQuantityTypeIdentifierDietaryProtein`
- `HKQuantityTypeIdentifierDietaryFatTotal`
- `HKQuantityTypeIdentifierDietaryCarbohydrates`

sourceName等を用いて、あすけん由来のデータだけを抽出する。

### 5.3 消費カロリー：Fitbit CSV

| 項目 | 要件 |
|---|---|
| データソース | Fitbit |
| 連携方式 | CSV手動アップロード |
| 取込項目 | 消費カロリー、歩数、アクティブ時間 |
| 頻度 | 必要時のみ |
| 補正 | 手動で補正係数を設定可能 |

Fitbitの消費カロリーは高めに出る可能性があるため、分析には補正後値を使う。

```text
補正後消費カロリー = Fitbit消費カロリー × Fitbit補正係数
```

初期値は `1.00`。設定画面で変更可能とする。

保存データは以下を持つ。

- 元の消費カロリー `raw_calories_kcal`
- 補正係数 `calorie_factor`
- 補正後消費カロリー `adjusted_calories_kcal`

### 5.4 筋トレ記録

| 項目 | 要件 |
|---|---|
| 記録方式 | アプリ内手入力 |
| セッション | 1回のトレーニング内に複数種目を登録 |
| 分割管理 | 胸、背中、肩、脚の日を登録 |
| セット種別 | ウォームアップ、メインセット |
| 有効セット | ウォームアップ以外、つまりメインセットをカウント |
| RPE/RIR | MVPでは記録しない |
| 痛み・違和感 | メモ欄で管理 |
| 種目マスタ | 主働筋のみ持つ |
| 部位カテゴリ | 胸、背中、肩、脚 |
| 部位別セット数 | 主働筋に1セット加算 |
| 自重/アシスト種目 | MVPでは通常の重量入力のみ |
| テニス/有酸素 | MVPでは筋トレ週次サマリーに含めない |

### 5.5 体型写真

| 項目 | 要件 |
|---|---|
| 対応フェーズ | MVPから対応 |
| 用途 | 見た目の変化を補助的に確認 |
| 保存方法 | 写真機能をMVPに入れる場合は、Supabase Storage等のクラウドストレージを限定利用 |
| 表示 | 日付別に一覧・比較できることが望ましい |
| セキュリティ | 写真も user_id とRLS相当で本人のみアクセス可能にする |

注意：元の取込ファイルは保存しない方針だが、写真管理にはストレージが必要である。そのため、Storageは「取込ファイル保存には使わないが、体型写真には使う」方針とする。

## 6. データ取込要件

### 6.1 取込対象

| データ種別 | MVP必須 | 取込方式 |
|---|---:|---|
| オムロン体組成CSV | ○ | ファイルアップロード |
| Apple Health ZIP | ○ | ファイルアップロード、ZIP/XML解析 |
| Fitbit CSV | ○ | ファイルアップロード |
| 筋トレ | ○ | 手入力 |
| 体型写真 | ○ | 写真アップロード |

### 6.2 取込前プレビュー

取込前に、可能な範囲で以下を表示する。

- 対象期間
- 取込予定件数
- 重複件数
- 上書き件数
- エラー件数
- 取込サンプル数行

### 6.3 再取込ルール

再取込時は、対象期間と取込モードを指定できる。

```text
対象期間：開始日〜終了日
取込モード：スキップ / 上書き
```

上書きの場合は、該当ユーザー、該当データ種別、該当期間の既存データを削除または更新してから、新データを保存する。

### 6.4 取込ログ

取込ログには以下を保存する。

- 取込日時
- データ種別
- ファイル名
- ファイルハッシュ
- 取込モード
- 対象期間
- 取込件数
- スキップ件数
- 上書き件数
- エラー件数
- エラー詳細
- プレビュー結果

元ファイル自体は保存しない。

### 6.5 エラー表示

MVPでは、エラー行と理由を表示する。将来はエラーCSVダウンロードに対応する。

### 表示内容

1. KPIカード、6枚以上
2. 体組成7日移動平均トレンド
3. 直近7日サマリー
4. 目標進捗
5. アラート要約
6. 改善アクション要約
7. 体組成グラフ
8. カロリー収支グラフ
9. PFCグラフ
10. 筋トレ成長グラフ

### KPI候補

- 体重7日平均
- 体脂肪率7日平均
- 体脂肪量7日平均
- 骨格筋量7日平均
- 直近7日カロリー収支
- 摂取カロリー平均
- タンパク質達成率
- 筋トレ回数
- 部位別セット充足率

## 7.3 体組成グラフ

| 項目 | 要件 |
|---|---|
| 表示期間 | 30日、90日、180日、全期間を切替 |
| 表示方式 | 当日値を薄く、7日移動平均を強調 |
| 対象指標 | 体重、体脂肪率、体脂肪量、骨格筋率、骨格筋量 |
| 判断軸 | 当日値ではなく7日移動平均を主にする |

## 7.4 カロリー収支

カロリー収支は以下に統一する。

```text
カロリー収支 = 摂取カロリー - 補正後消費カロリー
```

- マイナス：カロリー不足、減量方向
- プラス：カロリー余剰、増量方向

日次、週次、月次で表示する。

脂肪増減推定は参考値として表示する。

```text
脂肪増減推定kg = カロリー収支累計 / 7,700
```

7,700kcal/kgは初期値。将来設定可能にしてもよい。

## 7.5 PFC表示

PFCは以下をすべて表示する。

- g
- kcal
- 比率
- 目標比率との差分

タンパク質目標は、固定gと体重1kgあたりgの両方を表示する。

## 7.6 筋トレ記録

### 入力開始方法

以下すべてに対応する。

1. テンプレートを選んで開始
2. 空のセッションから開始
3. 前回と同じメニューから開始

### 入力画面

スマホ入力を重視し、以下を表示する。

- 種目リスト
- セット入力
- 前回記録
- 直近3回
- PR
- 前回比差分

セット入力は以下を重視する。

- 前回値コピー
- テンキー風入力UI
- セットごとの自動保存
- 最後に完了ボタン

### テンプレート

初期テンプレートは以下。

- 胸
- 背中
- 肩
- 脚

テンプレートには以下を保存する。

- 種目
- セット数
- 目標rep
- 目安重量
- 種目順

## 7.7 種目別成長

表示対象は以下。

- 推定1RM
- 最大重量
- 最大rep数
- 総ボリューム
- 直近10回推移

種目別成長のメイン指標は推定1RMとする。

```text
推定1RM = 重量kg × (1 + reps ÷ 30)
```

## 7.8 週次サマリー

週次サマリーはMVP必須。任意の日を基準に、直近7日間のローリング集計を表示する。

表示順は以下。

1. 体組成
2. 食事
3. 筋トレ
4. 改善アクション

表示項目は以下。

- 平均体重
- 平均体脂肪率
- 体脂肪量の変化
- 摂取カロリー平均
- PFC平均
- 筋トレ回数
- 部位別セット数
- 種目別成長
- 次週の改善アクション

## 7.9 アラート

アラート専用画面を作る。

### アラート対象

- 7日累計カロリー余剰または目標未達
- 脂質過多
- 体重7日平均増加
- 体脂肪量停滞
- 胸・背中・肩の部位別セット数不足
- タンパク質未達が3日以上

### 判定方針

| アラート | 判定 |
|---|---|
| 体重増加 | 設定画面で閾値変更可能 |
| 体脂肪停滞 | 体脂肪量kgで判定。期間・閾値は設定可能 |
| カロリー収支 | 設定画面で閾値変更可能 |
| 脂質過多 | 摂取カロリー超過時だけ脂質過多をチェック |
| タンパク質不足 | 目標未達が3日以上でアラート |
| 部位別セット数不足 | MVPは胸・背中・肩のみ対象 |

### 改善アクション

改善アクションは週次サマリーに表示する。MVPではルールベース、将来AIコメント化する。

改善アクションは以下の粒度とする。

- 食事：脂質を1日-10g、摂取カロリーを-150kcalなど数値で提示
- 筋トレ：胸の有効セットを+3セットなど具体的に提示

## 7.10 データ一覧・削除・再取込導線

データ一覧画面では、以下を扱う。

- 体組成
- 摂取カロリー/PFC
- 消費カロリー
- 筋トレ記録
- 取込ログ

操作は以下。

| データ種別 | 閲覧 | 削除 | 直接編集 | 修正方法 |
|---|---:|---:|---:|---|
| 体組成 | ○ | ○ | × | 再取込 |
| 摂取/PFC | ○ | ○ | × | 再取込 |
| 消費カロリー | ○ | ○ | × | 再取込 |
| 筋トレ記録 | ○ | ○ | ○ | 直接編集 |
| 取込ログ | ○ | 原則× | × | 参照 |

削除単位は以下すべてに対応する。

- 1レコード
- 日付単位
- 期間指定
- データ種別 + 期間指定

削除時はMVPでは確認ダイアログを表示する。
